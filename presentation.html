02 Die Idee

→
Persönlicher Anlass
Der Schneider/Amstrad PC1512-DD war der erste eigene Computer. Das Projekt ist ein Stück digitale Nostalgie — und ein komplettes Full-Stack-Weiterbildungsprojekt.
→
Technische Ambition
Ziel war kein CRUD-Tutorial, sondern ein vollständiger, produktionsfähiger Stack: eigene Domain, VPS, SSL, Docker, echte Auth mit E-Mail-Versand, und ein WebAssembly-Emulator.
→
Ergebnis
Eine öffentlich erreichbare Web-Plattform unter eigener Domain — entwickelt, deployt und betrieben von einer Person.
Der Computer
Schneider Amstrad PC1512-DD
Intel 8086 @ 8 MHz
512 KB RAM (erweiterbar auf 640 KB)
CGA-Grafik — 320×200, 4 Farben
2× 5,25″ Diskette — je 360 KB
MS-DOS 3.3 + GEM Desktop
Erscheinungsjahr: 1986

Docker Compose nginx Let's Encrypt Ubuntu 24.04
Produktions-Deployment via Docker Compose. SSL-Zertifikat automatisch über Certbot.

05 Registrierung & E-Mail-Bestätigung

1
Formular ausfüllen
Benutzername, E-Mail, Passwort. Passwort wird serverseitig mit bcrypt gehasht — niemals im Klartext gespeichert.
2
Verification Token
Backend generiert ein einzigartiges Token, speichert es in der DB mit 72h Ablaufzeit. Account ist zunächst unverified.
3
Bestätigungs-E-Mail
Mail wird über Brevo API versendet. Absender: noreply@retrokauz.de — eigene Domain, eigenes Hosting.
4
Klick & Freischaltung
Link aus der Mail → Backend setzt is_verified = true → Emulator-Zugriff freigeschaltet.
E-Mail-Dienst
Brevo (ehemals Sendinblue)
Transaktions-E-Mails via Brevo REST API. Kein SMTP auf Port 587 (war geblockt) — API-Aufruf direkt aus dem Backend.

Absender-Domain: retrokauz.de
SPF & DKIM: konfiguriert beim Hoster
E-Mail-Template: HTML im Retro-Design
Sicherheit
JWT-Authentifizierung
Nach Login: JWT-Token (HS256) im localStorage. Gültig 24h. Jede API-Anfrage schickt den Token im Authorization-Header.

06 Der Browser-Emulator

→
v86 — WebAssembly x86-Emulation
Open-Source-Projekt: vollständiger Intel x86-Prozessor in WebAssembly. Emuliert CPU, RAM, VGA, BIOS, Diskettenlaufwerke — direkt im Browser-Tab.
→
Nur für eingeloggte Nutzer
Der Emulator steckt hinter einer Protected Route. Wer nicht eingeloggt ist, wird zum Login weitergeleitet.
→
Komponenten im Frontend
EmulatorScreen.jsx: Screen-Container (div + canvas für CGA/VGA), Start-Button, Status-Anzeige, F11 = Vollbild.
→
Aktuelle Konfiguration
Startet FreeDOS. Disk-Images werden über nginx als statische Dateien ausgeliefert (/v86/). WASM-Datei: ~2 MB.
Technische Details v86
v86.wasm — Kern der Emulation (~2 MB)
seabios.bin — PC-BIOS (SeaBIOS)
vgabios.bin — VGA-BIOS
freedos722.img — Boot-Image (720 KB Diskette)
32 MB RAM emuliert, 2 MB VGA-Speicher
CGA-Modus — 320×200, 4 Farben
Das Emulatorfenster ist live erreichbar unter
1512.retrokauz.de/emulator — nach Login.

07 Infothek & Admin-Panel

Infothek
Artikel-System mit Markdown
Artikel werden im Admin-Bereich erstellt und mit Markdown verfasst. Das Frontend rendert sie mit react-markdown. Kategorisierung möglich (z. B. Geschichte, Technik, Software).

Öffentlich lesbar — kein Login erforderlich.
Technische Daten
Eigene Specs-Seite
Separate Seite mit Hardware-Spezifikationen: Prozessor, RAM, CGA-Grafik, Diskettenlaufwerke — mit historischen Erklärungen für Nicht-Techniker.
Admin-Panel
[ ADMIN PANEL ]
Zugang: Nur für Nutzer mit is_admin = true. Versteckter Link im Footer. Falsche Rolle → 403.
Artikel-Verwaltung:
  • Artikel erstellen mit Titel, Slug, Kategorie, Excerpt, Markdown-Body
  • Artikel bearbeiten (Slug bleibt unveränderlich)
  • Artikel löschen (mit Bestätigungs-Dialog)
Slug-Generierung: Automatisch aus Titel (Umlaute korrekt ersetzt: ä→ae etc.)

08 Deployment auf dem VPS

1
VPS eingerichtet
Ubuntu 24.04 LTS auf Contabo VPS. Docker + Docker Compose installiert. Firewall konfiguriert (Ports 80, 443 offen).
2
Domain & DNS
Domain retrokauz.de + Subdomain 1512.retrokauz.de → A-Record auf VPS-IP. DNS-Propagation abgewartet.
3
SSL mit Let's Encrypt
Certbot im Frontend-Container. Zertifikat automatisch ausgestellt und eingebunden. Auto-Renewal aktiv. Gültig bis Aug 2026.
4
Docker Compose Prod
3 Services: db (MySQL), backend (FastAPI), frontend (nginx + React + v86). Persistentes Volume für Datenbank.
5
Umgebungsvariablen
.env-Datei: DB-Passwort, JWT-Secret, Brevo-API-Key, E-Mail-Absender, App-URL. Nie im Git-Repo.
# Deployment-Befehl auf dem VPS $ cd /opt/1512dd $ git pull origin main $ docker compose \ -f docker-compose.prod.yml \ up -d --build # Drei Container starten: ✓ db (MySQL 8.0) ✓ backend (FastAPI :8000) ✓ frontend (nginx :80/:443) # nginx leitet weiter: / → React App (statisch) /api/ → FastAPI Backend /v86/ → WASM + Disk-Images

09 Was die Plattform bietet

  • ▶ Browser-Emulator (v86):Amstrad PC1512-DD läuft direkt im Browser. FreeDOS startet ohne Plugin oder Download. Vollbildmodus via F11.
  • ▶ Registrierung mit E-Mail-Bestätigung:Vollständige Auth-Strecke. Bestätigungsmail über Brevo API, Absender noreply@retrokauz.de (eigene Domain).
  • ▶ JWT-Login & geschützte Routen:Emulator nur für verifizierte Nutzer. Admin-Panel nur für Admins. Tokens sind 24h gültig.
  • ▶ Infothek mit Markdown-Artikeln:Kategorisierte Artikel, public zugänglich. Volltexte in Markdown, gerendert mit react-markdown.
  • ▶ Technische Daten-Seite:Hardware-Specs des PC1512 mit historischen Erläuterungen — Intel 8086, CGA, Diskettenlaufwerke.
  • ▶ Admin-Panel:Artikel erstellen, bearbeiten, löschen. Kategorie-Zuweisung, automatische Slug-Generierung.
  • ▶ HTTPS + eigene Domain:Let's Encrypt SSL, Auto-Renewal. 1512.retrokauz.de ist öffentlich erreichbar.
